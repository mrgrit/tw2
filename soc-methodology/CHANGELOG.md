## v0.2 — 2026-06-10 (cycle-002, baseline 전수 채굴 / deep-dive)

**관제한 데이터**: 동일 baseline(276) — 단, 증거 블롭을 *전수* 파싱(66 Suricata flow + Wazuh/명령).

**경험에서 나온 교훈 (cycle-001 교정 포함)**
- **#5 `.1` 은 게이트웨이다 (귀속 교정).** cycle-001 이 "insider 공격 호스트"라 한 `10.20.31.1` 은
  실은 세그먼트 게이트웨이. IPS 가 라우팅 후 last-hop G/W 를 src 로 기록 → src_ip = 유입 경로,
  호스트 신원 아님. 호스트 식별엔 명령 로그(2202/2203)+시간 상관 필요.
- **#6 타깃측 탐지가 1차 진실.** 캡처된 명령(2203)은 `-sT :80` 1종뿐인데 IPS 는 5개 타깃
  SYN 스윕을 관측. insider(2202) 명령 미수집이 상관 공백의 근본 원인 → 명령 로그는 보조.
- **단일 호스트 → 다중 호스트 스윕.** 정찰 표적: portal(8000)·HTTPS(443)·**SIEM 콘솔(5601)**.
- **신규 신호 발굴**: Wazuh rule 100250 HAProxy de-NAT → LAN 관리자 192.168.0.107 식별(양성).
  cycle-001 은 이 28건을 통째로 놓쳤다 → evidence 전수 파싱의 가치 입증.
- **dedup 키 실증**: SQLi 1건이 4개 코호트 교차 적재 → `(rule_id, alert_ts)` 로 통합. cycle-001 의
  "5 battle" 은 battle 행 기준 과대계상이었음(고유 코호트 4).

**방법론 반영**: §3 세그먼트 맵 교정+admin 화이트리스트, §4 dedup 키 갱신, §4.5 **증거 블롭 파서 규약** 신설.

## v0.1 — 2026-06-10 (cycle-001, baseline)

**관제한 데이터**: 2026-06-03 ~ 06-06, 276 이벤트(5개 교과목 인덱스).

**경험에서 나온 교훈**
- **#1 탐지는 alert가 아니라 evidence에 있다.** `kind=alert` 는 5건뿐이지만 실제 Suricata/Wazuh
  탐지는 `mission_check.payload.evidence` 블롭(log_contains 118 + wazuh_alert 68)에 박혀 있었다.
  → 관제 에이전트의 1차 파싱 대상은 evidence 블롭.
- **#2 source IP 세그먼트 귀속이 최고가치 트리아지 키.** 같은 nmap 스캔이 외부(10.20.20.x)냐
  내부(10.20.31.x)냐에 따라 인시던트 성격이 완전히 달라진다. AI 채점도 동일 기준으로 판정 중이었다.
- **#3 코호트 교차 중복.** 단일 SQLi 경보가 5개 코호트 인덱스에 동시 적재 → dedup 키 필요.
- **#4 휘발성 비중이 크다.** volatile=true 186/246 → 캡처 시점 보존이 사후 분석의 전제.

**다음 사이클 액션**
- 신규 이벤트 델타만 보는 증분 사이클 검증(현재 06-06 이후 데이터 없음 → 인입 대기).
- evidence 블롭에서 src_ip/dest/signature 를 뽑는 경량 파서 규약화(v0.2).

**미해결**
