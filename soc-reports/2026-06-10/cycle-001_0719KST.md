## 2. 탐지된 위협 · 인시던트

### INC-001 — 내부 세그먼트發 nmap SYN 스캔 (진행/반복)

- **시그니처**: Suricata `signature_id=1000005` "6V6 Possible nmap SYN scan",
  category *Attempted Information Leak*, severity 2 → Wazuh `rule 86601` lvl 3 (agent=ips) 로 미러링.
- **플로우**: `src_ip 10.20.31.1` → `dest_ip 10.20.32.80 : 443/tcp` (pkts_toserver=1, SYN only).
- **관측 시각**: 2026-06-06 13:09Z, 13:33Z (반복). 학생 셸 로그에도 동시간대
  `nmap -sT -p80 10.20.32.80` (attacker-ext, port 2203) 존재.
- **연관 증거**: `mission_check` log_contains(red-1/red-2) 와 wazuh_alert(blue-4) 가
  동일 플로우를 가리킴 — 공격·탐지 양쪽이 같은 이벤트를 포착.

> **분석가 판단**: 타깃(10.20.32.80)에 실제로 SYN 스캔이 도달했고 IPS가 탐지했다.
> 단, 패킷의 source 가 **10.20.31.1(내부 attacker 세그먼트)** 라는 점이 핵심
> (→ §3 귀속 분석). 외부 정찰이 아니라 **내부 발판에서의 east-west 정찰**로 분류해야 한다.

### INC-002 — SQL Injection (UNION SELECT) 탐지

- **시그니처**: Suricata/Wazuh `rule 86601` lvl 3 "6V6 SQL Injection - UNION SELECT" (agent=ips).
- **관측 시각**: 2026-06-03 19:24:11Z.
- **확산**: 동일 경보가 **5개 battle/코호트**(네트워크보안및응용·공격기법·보안운영입문·보안운영)에

- **상태**: 페이로드 본문/대상 URL 미수집 → UNION SELECT 시도가 차단/허용됐는지 SIEM 단독으로는 불명.

## 3. 출처(Source IP) 귀속 분석 — 이번 사이클의 핵심

6v6 망 구조상 공격자는 두 페르소나가 있다(`CLAUDE.md`):

| 세그먼트 | 정체 | SOC 해석 |
|----------|------|----------|
| `10.20.20.x` (WAN, 예 .202) | **attacker-ext (outsider)** | 망 외부 침입자 — perimeter 공격 |
| `10.20.31.x` (예 .1) | **attacker (insider)** | 내부 발판 — post-foothold/lateral |
| `10.20.32.x` (예 .80) | 타깃/DMZ 서비스 | 피격 대상 |

INC-001 의 스캔은 `10.20.31.1`(insider) 에서 출발했다. AI 채점 verdict 도 동일하게
"attacker-ext(10.20.20.202) 트래픽은 Suricata 0건, 스캔 출처는 내부 대역"이라 판정했다.

> **관제 함의**: 동일한 "nmap 스캔"이라도 **외부 침입자의 최초 정찰**인지 **내부 침해 후
> 측면 정찰**인지에 따라 대응 우선순위와 IR 시나리오가 완전히 달라진다. source IP 세그먼트
> 귀속은 본 환경에서 **가장 신호 대비 가치가 높은 1차 트리아지 키**다.

## 5. 권고 대응 (real-SOC 관점)

1. **INC-001 트리아지**: `10.20.31.1` 의 타깃 접근을 *내부 측면 정찰*로 분류하고, 해당 발판이
   인가된 것인지(실습 설계상 의도된 insider 인지) 확인. 외부 페르소나로 수행됐어야 하는
   미션이면 공격 절차 오류로도 기록.
2. **경보 승격(G-1)**: lab_monitor 수집 시 Suricata/Wazuh 경보를 `kind=alert` 1급 문서로
   적재(현재 mission_check 증거에만 존재). *코드 변경 제안일 뿐, 본 에이전트는 미적용.*
3. **방어 룰 검증(G-3)**: 블루팀 `local_rules.xml` 의 기대 커스텀 룰 배포 여부 점검.
4. **상관 타임라인**: 2203 정찰 명령 ↔ IPS SYN 스캔 경보를 시각·src 로 묶어 단일 공격 세션 타임라인 구성.
5. **dedup(G-5)**: `(rule_id, alert.ts, src_ip, dest_ip)` 를 인시던트 동일성 키로 사용.
