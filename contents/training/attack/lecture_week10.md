# 공격기법 W10 — 필터를 빠져나가기: 인코딩·분할·난독화 우회 vs 정규화 방어

> 공격기법 트랙 10주차. 선행: W04–W05(SQLi/XSS·WAF). 인프라: el34 (ModSec CRS). 플랫폼: tw2.

---

## 1. 이번 주의 통찰 — 같은 공격, 다른 표현

WAF는 알려진 패턴을 찾는다. 공격자는 **같은 의미를 다르게 표현**해 패턴 매칭을 피한다 — 인코딩, 분할,
난독화. 방어자는 **정규화(normalization)** 로 표현을 통일한 뒤 검사한다. 이 둘의 싸움이 W10이다.

```
 공격: 'UNION SELECT' → 'UN/**/ION%20SE%2fLECT' → 'UNION' …(같은 의미, 다른 표현)
 방어: 디코드·주석제거·소문자화(정규화) → 'union select' → 매칭
```

---

## 2. 인코딩 우회

| 기법 | 예 |
|------|----|
| URL 인코딩 | `%55NION` (U) |
| 이중 인코딩 | `%2555` (%25=%, %55=U → %U → U) |
| 유니코드/와이드 | `%u0055`, overlong UTF-8 |
| HTML 엔티티 | `&#85;NION` |

```bash
docker exec el34-attacker sh -c "curl -s -o /dev/null -w 'double=%{http_code}\n' -H 'Host: dvwa.el34.lab' 'http://10.20.30.1/?id=1%2527%20UNION'"
```

---

## 3. 분할·난독화

- **주석 분할**: `UN/**/ION SE/**/LECT` (SQL 주석으로 키워드 쪼개기).
- **공백 대체**: `UNION%09SELECT`(탭), `%0a`(개행), `/**/`.
- **대소문자**: `UnIoN sElEcT`.
- **파라미터 오염(HPP)**: `?id=1&id=UNION` (서버별 병합 차이 악용).

---

## 4. 정규화 방어 — CRS가 이기는 이유

ModSec CRS는 검사 전에 **transformation(t:)** 으로 정규화한다:
- `t:urlDecodeUni`(URL/유니코드 디코드), `t:lowercase`, `t:removeComments`, `t:compressWhitespace`.
- 그래서 인코딩/주석/대소문자 우회는 **정규화 후 같은 패턴**이 돼 잡힌다 → 대부분 여전히 403.
- 우회가 통하려면 **정규화가 놓치는 표현**(특정 인코딩/컨텍스트)을 찾아야 — 점점 어렵다.

## 5. 실습(lab) 형식 — 8 미션

1. **점검**: 대상
2. **기준 공격**: 평문 SQLi → 403(baseline)
3. **인코딩 우회**: 이중/유니코드 → 결과
4. **분할·난독화**: 주석/공백/대소문자 → 결과
5. **정규화 확인**: 대부분 여전히 403(CRS 정규화)
6. **탐지 분석**: ModSec 942 + 정규화 흔적
7. **우회 한계 + 고급**: 정규화가 놓치는 표현
8. **필터 우회 보고서**

> 명령은 el34 호스트(`ssh ccc@192.168.0.151`)에서 `docker exec el34-attacker` 로. 인가된 실습만.
