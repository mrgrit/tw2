# 공격기법 W08 — 중간고사 CTF: 익스플로잇 체인으로 플래그(셸)를 캡처하기

> 공격기법 트랙 8주차 (중간 평가). 선행: W01–W07. 인프라: el34. 플랫폼: tw2.

---

## 1. 이 시험이 평가하는 것

W01–W07의 개별 기법(정찰·API/JWT·SQLi·XSS·접근제어·SSRF)을 **하나의 익스플로잇 체인**으로 엮어
**플래그(코드 실행 거점=셸)** 를 캡처한다. 단일 취약점이 아니라 **체인을 구성하는 능력**을 본다.

```
 정찰 → 표면 식별 → 웹 익스플로잇(침투) → 발판 → 셸 획득(플래그 캡처)
 (W01-02)  (W03-07)    (SQLi/XSS/...)      (W06)    (포트 48008 = 캡처된 셸)
```

CTF의 핵심: "어느 취약점으로 들어가서, 어떻게 코드 실행까지 가나"의 경로(kill chain).

---

## 2. 익스플로잇 체인의 단계

1. **정찰**: 포트/서비스/엔드포인트 매핑(W01-02).
2. **취약점 식별**: 어느 입력이 익스플로잇 가능한가(W03-07).
3. **침투**: 가장 효과적인 취약점으로(SQLi 인증우회/RCE/업로드).
4. **셸 획득**: webshell/리버스셸로 코드 실행 거점 확보(플래그).
5. **정리**: 흔적 최소화(고급).

방어자는 이 체인을 WAF/IDS/호스트 로그로 **역재구성**한다.

---

## 3. 플래그 = 셸 (코드 실행 거점)

CTF의 "플래그"는 보통 **코드 실행을 증명하는 것** — 셸 획득, 특정 파일 읽기, 리스너 포트. 이번 시험은
**리스너 포트 48008**(캡처된 셸)을 플래그로 한다.

```bash
# 셸 캡처(플래그) — 리스너 포트 48008
docker exec el34-web sh -c 'nohup python3 -m http.server 48008 >/dev/null 2>&1 &'
docker exec el34-web osqueryi --json 'SELECT pid,port FROM listening_ports WHERE port=48008;'
```

---

## 4. 방어자 — 체인 역재구성 + 탐지 룰

방어자는 흩어진 흔적(WAF 942/913, IDS, 호스트 포트)을 한 체인으로 엮고, 정찰을 잡는 IDS 룰(sid 9408001)을
쓴다(W03 secuops 패턴).

---

## 5. 실습(CTF lab) 형식 — 9 미션

1. **점검**: 도구 + 대상
2. **정찰**: 표면 식별
3. **웹 익스플로잇**: 침투(SQLi/스캐너)
4. **셸 캡처(플래그)**: 리스너 48008 → osquery 확인
5. **IDS 탐지 룰**: 정찰 탐지(sid 9408001)
6. **체인 역재구성**: WAF/IDS/호스트 흔적 종합
7. **플래그 검증**: 48008 캡처 확인
8. **CTF 보고서**: 익스플로잇 체인
9. **정리**: 셸/룰 self-clean

> 명령은 el34 호스트(`ssh ccc@192.168.0.151`)에서. 셸/룰은 self-clean(공유 인프라 보존). 인가된 실습만.
