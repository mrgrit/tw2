# agent-ir-adv W07 — Multi-stage 피싱→웹셸→측면이동: Human+Agent 하이브리드

> **본 주차의 한 줄 요약**
>
> W07은 여러 단계를 잇는 **다단계 하이브리드 공격**을 다룬다. 전형적 킬체인: ① **피싱**(사람 대상 사회공학)으로
> 초기 자격·접근 획득, ② **웹셸**(웹 서버에 심는 백도어 스크립트)로 발판 확보, ③ **측면이동**(agent-ir W05)으로
> 내부 확산. "하이브리드"란 **사람과 AI 에이전트가 역할을 나눠** 협업한다는 뜻이다: 피싱 메일 작성·표적 선정은
> AI가 **대량·정교하게**(딥페이크 수준 문장), 초기 클릭 유도는 사람 심리 악용, 웹셸 이후 내부 탐색·이동은 AI가
> **자동·고속**으로. 방어의 핵심은 **단계 간 연결**을 보는 것 — 피싱 하나, 웹셸 하나는 각각 탐지 대상이지만,
> **"피싱으로 들어온 자격이 웹셸을 심고 그 웹셸이 측면이동을 했다"** 는 **연결된 이야기**로 보면 전체 침해가
> 드러난다. 각 단계 탐지: 피싱(의심 발신·링크·첨부), 웹셸(웹 디렉토리의 비정상 스크립트·비정상 프로세스 실행),
> 측면이동(비정상 내부 연결·인증). el34 웹으로 웹셸 흔적(비정상 스크립트·프로세스)을 관찰한다.
>
> **한 줄 결론**: 하이브리드 다단계 공격은 피싱→웹셸→측면이동을 사람+AI가 나눠 수행한다. 방어는 각 단계 탐지 +
> **단계 간 연결**(하나의 침해 이야기)로 전체를 본다. 연결이 개별보다 강력하다.

## 학습 목표

본 주차 종료 시 학생은 다음 5가지를 **본인 손으로** 할 수 있어야 한다.

1. 피싱→웹셸→측면이동 **다단계 킬체인**을 설명한다.
2. **피싱 지표**를 탐지한다(PHISHING_DETECTED).
3. **웹셸**(비정상 스크립트·프로세스)을 탐지한다(WEBSHELL_DETECTED).
4. 단계 간 **연결**로 침해 이야기를 구성한다(HYBRID_CHAIN).
5. Human+Agent 하이브리드의 역할 분담을 설명한다.

> **이 주차의 시선** — 흩어진 단계를 하나의 침해 이야기로 잇는다.

## 0. 용어 해설 (하이브리드 공격)

| 용어 | 영문 | 뜻 | 비유 |
|------|------|----|------|
| **피싱** | Phishing | 사회공학 초기 침투 | 미끼 편지 |
| **웹셸** | Web Shell | 웹 서버 백도어 스크립트 | 숨은 뒷문 |
| **하이브리드** | Hybrid | 사람+AI 협업 공격 | 합동 작전 |
| **초기 접근** | Initial Access | 최초 침투 | 첫 발판 |
| **단계 연결** | Stage Linking | 단계 간 인과 연결 | 이야기 잇기 |

> **헷갈리기 쉬운 한 쌍** — *개별 탐지* 는 "각 단계를 따로 잡음", *단계 연결* 은 "인과로 이어 전체 침해를 봄"이다.
> 후자가 하이브리드 공격의 답.

## 0.5 핵심 개념

### 0.5.1 다단계 하이브리드 킬체인

```mermaid
graph LR
    P["① 피싱<br/>(사람 심리 + AI 문장)"] --> C["자격/접근 획득"]
    C --> W["② 웹셸<br/>(웹서버 백도어)"]
    W --> L["③ 측면이동<br/>(AI 자동 확산)"]
    style P fill:#d29922,color:#fff
    style W fill:#f85149,color:#fff
```

사람(피싱 클릭 유도)과 AI(문장 생성·내부 자동 탐색)가 **역할을 나눠** 잇는다. 각 단계가 흔적을 남긴다.

### 0.5.2 피싱 탐지 — 초기 침투 차단

피싱 지표: **의심 발신**(도메인 사칭·신규 도메인), **의심 링크**(리다이렉트·유사 도메인), **의심 첨부**(매크로·
실행 파일), **긴급성 문구**(심리 압박). AI가 만든 피싱은 문장이 자연스러워 내용만으론 어렵다 — **기술 지표**
(발신·링크·첨부)를 함께 본다.

### 0.5.3 웹셸 탐지 — 발판 찾기

웹셸은 웹 디렉토리에 심은 **백도어 스크립트**(예: `.php`/`.jsp`가 명령 실행). 탐지: (1) **비정상 파일**(웹
디렉토리에 새 스크립트·비정상 이름·난독화), (2) **비정상 프로세스**(웹 서버가 셸·명령 실행), (3) **비정상
요청**(웹셸에 명령 전달하는 요청 패턴). el34 apache 로그·프로세스로 관찰.

### 0.5.4 단계 연결 — 침해 이야기

피싱·웹셸·측면이동을 **인과로 연결**한다: "10:00 피싱 클릭(user X) → 10:05 X 자격으로 웹서버 접근 → 10:06
웹셸 생성 → 10:10 웹셸이 내부 연결". 시간·주체·연쇄로 이으면 **전체 침해 이야기**가 나온다(agent-ir W08 체인
재구성의 하이브리드판). 개별 경보를 이야기로 묶어야 대응이 완전해진다.

### 0.5.5 Human+Agent 역할 분담

하이브리드에서 AI는 **규모·속도·정교함**을, 사람은 **심리·판단·유연성**을 맡는다: AI가 표적 1000명에게 맞춤
피싱을, 사람이 걸린 소수를 정밀 공략, AI가 내부를 자동 탐색. 방어도 대칭으로 — 각 단계에 맞는 탐지(피싱 필터·
웹셸 스캔·이동 탐지)를 겹층으로, 그리고 연결로 종합.

## 2. 흔한 오해·블루팀 노트

- **"AI 피싱은 내용으로 구분"** — 문장이 자연스럽다. 기술 지표(발신·링크·첨부)를 봐야.
- **"웹셸은 안티바이러스가"** — 난독화·신종은 우회. 비정상 파일·프로세스·요청 행위로.
- **"각 단계 탐지면 충분"** — 연결해야 전체 침해가 보인다. 개별은 부분.
- **관제 관점** — 피싱 필터·웹셸 스캔·이동 탐지가 겹층인지, 단계가 연결(같은 자격·시간·연쇄)되는지, 초기 접근
  (피싱)에서 잡는지 점검한다. 다단계 공격은 연결로 완성 대응.
