# 웹취약점 W15 — 기말: 종합 웹 침투 점검 (정찰→침해→보고 전 과정)

> 웹취약점 트랙 15주차 (기말 평가). 선행: W01–W14. 인프라: el34 (juice/dvwa). 플랫폼: tw2.
> 표준: WSTG(점검) + CVSS(평가) + 전문 보고.

---

## 1. 이 시험이 평가하는 것

한 학기(W01~W14)의 전 역량을 **하나의 종합 침투 점검**으로 통합한다 — 정찰부터 발견·입증·위험 평가·
전문 보고까지 실제 점검 한 사이클을 수행한다.

```
 종합 점검 킬체인
 ① 정찰(HTTP/기술/엔드포인트) → ② 인증(SQLi 우회/JWT) → ③ 주입(SQLi/XSS/RCE)
 → ④ 접근제어(A01/IDOR) → ⑤ 통합 탐지(WAF) → ⑥ 위험 평가(CVSS) → ⑦ 전문 보고
```

## 2. 평가 항목 (WSTG 한 바퀴)

1. **정찰**: 기술 핑거프린팅(nuclei), 엔드포인트 매핑(공개/보호).
2. **인증**: 로그인 SQLi 우회, JWT 분석.
3. **주입**: SQLi/XSS/RCE 시도와 WAF 탐지.
4. **접근제어**: 무인증 관리, 강제 브라우징.
5. **정보 노출**: DB 에러/배너.
6. **위험 평가**: 발견을 CVSS로 정량화.
7. **전문 보고**: Executive Summary + 발견 + 권고.

## 4. 트랙 총정리

W01~W15로 웹 취약점 점검의 전 영역을 익혔다 — HTTP/정보수집, 인증/세션, SQLi/XSS/RCE, 접근제어,
전송/정보 노출, API, 자동화 스캐닝, 위험 평가/보고. **수동의 정밀 + 자동의 속도 + 위험 기반 우선순위 +
재현 가능한 보고** 가 전문 웹 보안 점검자의 핵심이다.
