# 웹취약점 W14 — 위험 평가(CVSS)와 전문 보고서

> 웹취약점 트랙 14주차. 선행: W01–W13. 인프라: el34. 플랫폼: tw2. 표준: CVSS 3.1, OWASP Risk Rating.

---

## 1. 이번 주의 통찰 — 발견은 절반, 전달이 나머지 절반

취약점을 찾는 것(W01~W13)만으로는 조직이 움직이지 않는다. **위험을 정량화(CVSS)하고 우선순위를 매겨
의사결정자가 행동할 수 있게 전달**해야 점검이 가치를 가진다. 같은 SQLi라도 인증 뒤·내부망이면 위험이
다르다 — 맥락이 점수를 바꾼다.

```
 점검 → [위험 평가] → 보고
        ① CVSS 벡터/점수  ② 심각도 분류  ③ 우선순위(가능성×영향)  ④ 권고
```

---

## 2. CVSS 3.1 기본 메트릭

```
CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H  → 9.8 (Critical)
 AV(공격벡터) N네트워크/A인접/L로컬/P물리   AC(복잡도) L낮음/H높음
 PR(권한) N없음/L낮음/H높음                  UI(상호작용) N불필요/R필요
 S(범위) U불변/C변경                          C/I/A(기밀/무결/가용) H높음/L낮음/N없음
```
- 예) 무인증 원격 SQLi → AV:N AC:L PR:N UI:N C:H I:H A:H = **9.8 Critical**.
- 예) 정보 노출(배너) → C:L 나머지 약함 = **5.3 Medium** 수준.

---

## 3. 심각도 등급 (CVSS 점수 → 라벨)

```
 9.0–10.0 Critical | 7.0–8.9 High | 4.0–6.9 Medium | 0.1–3.9 Low | 0.0 None
```

---

## 4. 우선순위 — 가능성 × 영향

CVSS 외에 **악용 가능성(공개 익스플로잇·인증 필요 여부)** 과 **자산 중요도(데이터 민감도)** 를 곱해
수정 우선순위를 정한다. Critical+쉬운 악용+민감 데이터 = 최우선.

---

## 5. 전문 보고서 구조

```
 1. Executive Summary   (비전문가용: 위험 한 문단 + 등급 분포)
 2. 점검 범위/방법론     (대상, WSTG/PTES, 기간)
 3. 발견 사항(상세)      (제목/심각도/CVSS/증거(PoC)/영향/재현/권고)
 4. 위험 매트릭스        (가능성×영향)
 5. 권고 요약            (우선순위별 조치)
```
- 각 발견: **재현 가능한 증거(PoC) + 영향 + 구체 권고**. "있다"가 아니라 "이렇게 재현되고, 이렇게 막는다".
