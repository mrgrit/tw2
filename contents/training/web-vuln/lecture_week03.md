# 웹취약점 W03 — 정보수집(Recon): WSTG-INFO 정찰 vs 정보 노출 축소

> 웹취약점 트랙 3주차. 선행: W01–W02. 인프라: el34. 플랫폼: tw2. 방법론: OWASP WSTG-INFO.

---

## 1. 이번 주의 통찰 — 노출된 정보가 공격을 연다

WSTG-INFO 단계 — 앱이 **의도치 않게 노출한 정보**를 모은다. robots.txt, HTML 주석, 백업 파일, 버전 관리
디렉토리(.git), 에러 메시지. 작은 정보가 익스플로잇의 실마리가 된다.

```
 정보 노출원: robots.txt(숨긴 경로 역설적 노출) / HTML 주석(개발자 메모) /
              백업파일(.bak ~ .old) / .git(소스 전체) / 에러 메시지(경로·버전)
```

---

## 2. 노출 정보 수집

```bash
curl -s -H 'Host: dvwa.el34.lab' http://10.20.30.1/robots.txt        # 숨긴 경로
curl -s -H 'Host: dvwa.el34.lab' http://10.20.30.1/ | grep -i '<!--'  # HTML 주석
curl -sI -H 'Host: dvwa.el34.lab' http://10.20.30.1/index.php.bak     # 백업 파일
curl -sI -H 'Host: dvwa.el34.lab' http://10.20.30.1/.git/config       # VCS 노출
```
- **robots.txt**: 숨기려던 경로(/admin)를 역설적으로 알려준다.
- **백업 파일**: `index.php.bak`/`~`/`.old`에 소스/설정 노출.
- **.git**: 노출 시 전체 소스 복원 가능(매우 위험).

---

## 3. 메타데이터·에러

- 응답 헤더/메타(생성 도구, 버전).
- 에러 메시지(stack trace) → 내부 경로/프레임워크/DB 노출(W11에서 심화).

---

## 4. 방어 — 노출 축소

- robots.txt에 민감 경로 적지 말 것(접근 제어로 보호).
- 백업/임시 파일 웹루트 밖, .git 웹 노출 차단.
- 에러 일반화, 주석/메타 제거(빌드 시).

---
