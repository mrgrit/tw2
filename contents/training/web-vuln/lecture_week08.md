# 웹취약점 W08 — 중간고사: JuiceShop 종합 점검(W01~W07) vs 통합 탐지·보고

> 웹취약점 트랙 8주차 (중간 평가). 선행: W01–W07. 인프라: el34 (juiceshop/dvwa). 플랫폼: tw2.

---

## 1. 이 시험이 평가하는 것

W01–W07의 점검 역량(HTTP·스캐너·정보수집·인증·SQLi·XSS·RCE)을 **하나의 앱(JuiceShop)에 종합 적용**해
취약점을 발견·입증하고 통합 보고한다. WSTG 방법론을 한 바퀴 돈다.

```
 종합 점검 흐름
 ① HTTP/정보수집 → ② 인증/세션 → ③ 입력 취약점(SQLi/XSS/RCE) → ④ 접근제어 → ⑤ 통합 보고
```

---

## 2. WSTG 종합 점검 순서

1. **정찰**: HTTP 메서드/헤더, robots/백업, 핑거프린팅.
2. **인증**: 로그인 SQLi 우회, JWT 분석.
3. **입력**: SQLi/XSS/경로순회 점검.
4. **접근제어**: 강제 브라우징(/ftp), 무인증 관리 엔드포인트.
5. **통합 보고**: 발견 취약점을 심각도·증거와 함께.

---

## 3. 실습(중간고사 lab) 형식 — 10 미션

1. **점검**: JuiceShop
2. **HTTP/정보수집**: 메서드/robots
3. **인증 우회**: 로그인 SQLi → JWT
4. **JWT 분석**: payload 노출
5. **입력 취약점**: SQLi/XSS
6. **접근제어**: 강제 브라우징(/ftp), 무인증 관리
7. **탐지 통합**: WAF 흔적 종합
8. **심각도 평가**: 발견 취약점 등급
9. **종합 점검 보고서**
10. **권고**: 방어 우선순위

> 명령은 el34 호스트(`ssh ccc@192.168.0.151`)에서 `docker exec el34-attacker` 로. 인가된 실습만.
