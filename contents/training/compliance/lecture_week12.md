## 1. 이번 주의 통찰 — 개인정보는 빌린 것이다

개인정보는 조직 소유가 아니라 **정보주체에게 빌린 것**이다. 수집·이용·보관·파기 전 과정에 법적 의무가
따른다. 개인정보보호법/GDPR은 **처리 원칙·정보주체 권리·안전성 확보 조치**를 요구하며, 위반은 거액의
과징금(GDPR 최대 전세계 매출 4%)으로 이어진다.

```
 생애주기: 수집 → 이용/제공 → 보관 → 파기  (각 단계에 동의·목적·최소화·안전조치)
```

  도메인·식별자 결합으로 **재식별 위험** = 개인정보 보호 결함.

## 3. 개인정보 처리 원칙

```

 정확성:      최신·정확하게 유지
 보유 제한:   목적 달성 시 지체없이 파기
 안전성:      암호화·접근통제·로깅
 투명성:      처리방침 공개, 동의
```

## 5. 가명·익명 처리, 마스킹

```
 가명처리: 추가정보 없이는 식별 불가(분석 활용 가능)
 익명처리: 복원 불가(개인정보 아님)
 마스킹:   화면/응답에 부분 가림 — 단, 부분 마스킹은 재식별 위험 잔존
```
