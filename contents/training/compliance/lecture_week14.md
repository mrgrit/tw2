# 컴플라이언스 W14 — 감사와 증적 수집

> 컴플라이언스 트랙 14주차. 선행: W01–W13. 인프라: el34. 플랫폼: tw2. 표준: ISO 19011(감사), ISMS-P 심사.

---

## 1. 이번 주의 통찰 — 증명할 수 없으면 하지 않은 것이다

감사·인증심사에서는 "우리는 통제하고 있다"는 말이 아니라 **재현 가능하고 무결성이 보장된 증적**이
필요하다. 통제가 아무리 잘 돼 있어도 증적이 없으면 부적합(NC) 판정을 받는다.

```
 감사 흐름: 계획 → 증적 수집 → 적합/부적합 판정 → 시정조치(CAPA) → 후속 확인
```

---

## 2. 증적 수집 (재현+무결성+추적)

설정/로그를 수집할 때 **해시(무결성) + 수집 시각/대상(추적)** 을 함께 남긴다.

```bash
docker exec el34-web sh -c "sha256sum /etc/apache2/conf-available/security.conf; date -u +%Y-%m-%dT%H:%M:%SZ"
```
- 출력: 파일 해시 + UTC 시각. 같은 명령으로 재현되고, 해시로 변조 여부를 검증 = 감사에서 인정되는 증적.

---

## 3. 증적의 3요건

```
 재현성: 같은 점검을 반복하면 같은 결과(명령·절차 명시)
 무결성: 해시/서명으로 수집 후 변조 없음 증명
 추적성: 누가·언제·어디서·무엇을 수집했는지 메타데이터
```

---

## 4. 내부감사 vs 인증심사

```
 내부감사: 조직 자체 점검(연 1회+), 개선 목적
 인증심사: 외부 심사기관(ISMS-P/ISO) — 최초/사후/갱신 심사
 둘 다 객관성(독립성) + 증적 기반 판정
```

---

## 5. 부적합(NC) 관리

```
 부적합 발견 → 원인 분석(근본원인) → 시정조치(CAPA) → 증적과 함께 조치 완료 → 후속 심사 확인
 중부적합(Major)/경부적합(Minor) 구분, 기한 내 조치 의무
```

## 6. 실습(lab) 형식 — 8 미션

1. **점검**: 대상
2. **증적 수집**: 해시+시각
3. **증적 3요건**: 재현/무결성/추적
4. **내부감사 vs 인증심사**
5. **감사 계획**: 범위/기준
6. **부적합 관리**: CAPA
7. **방어**: 감사 체계
8. **감사 보고서**

> 명령은 el34 호스트(`ssh ccc@192.168.0.151`)에서 `docker exec el34-X` 로. 신규 설치 없음.
