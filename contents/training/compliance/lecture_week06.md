# 컴플라이언스 W06 — 로깅·모니터링 컴플라이언스 (감사 추적)

> 컴플라이언스 트랙 6주차. 선행: W01–W05. 인프라: el34 (web/siem). 플랫폼: tw2. 표준: PCI-DSS 10, ISMS-P 2.9.

---

## 1. 이번 주의 통찰 — 로그 없는 보안은 사후 확인 불가

침해는 막지 못할 수 있다. 그러나 **무슨 일이 있었는지 추적 불가능**하면 대응·법적 대응·재발 방지가
모두 무너진다. PCI-DSS 요구사항 10은 **모든 접근에 감사 추적(audit trail)** 을 의무화한다.

```
 로그 생성(누가/언제/무엇을/결과) → 중앙 수집 → 무결성 보호 → 보존(1년+) → 모니터링/알림
```

---

## 2. 로그 활성 점검

```bash
docker exec el34-web sh -c "ls /var/log/apache2/*access*.log | wc -l"
```
- el34-web: vhost별 access/error 로그 다수 활성 = 웹 접근 감사 추적 존재(PCI 10.2 접근 기록).

---

## 3. 중앙 수집 (SIEM)

```bash
docker exec el34-siem sh -c "tail -1 /var/ossec/logs/alerts/alerts.json"
```
- el34-siem(Wazuh)가 IPS/web 로그를 중앙 수집·상관·알림 → 분산 로그를 한 곳에서 감사. **중앙 수집은
  로그 변조·삭제 방지와 통합 모니터링의 전제**.

---

## 4. 감사 추적 요소 (PCI-DSS 10.2)

```
 누가(user) · 언제(timestamp) · 무엇을(event/object) · 어디서(source IP) · 결과(success/fail)
 필수 기록: 인증 시도, 권한 변경, 관리 행위, 데이터 접근, 로그 자체 접근
```

---

## 5. 로그 무결성·보존·시간

```
 무결성: 로그 변조 방지(append-only, 해시, WORM), 로그 접근도 로깅
 보존: PCI 최소 1년(즉시 조회 3개월), 법규별 상이
 시간: NTP 동기화(PCI 10.4) — 시계 어긋나면 사건 시간선 붕괴
```

---

## 6. 실습(lab) 형식 — 8 미션

1. **점검**: 대상
2. **로그 활성**: Apache access 로그
3. **중앙 수집**: Wazuh 적재
4. **감사 추적 요소**: 누가/언제/무엇을/결과
5. **무결성·보존**: WORM/1년
6. **시간 동기화**: NTP
7. **방어**: 감사 추적 강화
8. **로깅 컴플라이언스 보고서**

> 명령은 el34 호스트(`ssh ccc@192.168.0.151`)에서 `docker exec el34-X` 로. 신규 설치 없음.
