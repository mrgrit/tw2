# 컴플라이언스 W03 — 자산 식별과 분류

> 컴플라이언스 트랙 3주차. 선행: W01–W02. 인프라: el34. 플랫폼: tw2. 표준: ISMS-P 2.1, ISO 27001 A.5.9.

---

## 1. 이번 주의 통찰 — 모르는 것은 지킬 수 없다

보호 대상을 모르면 통제를 배치할 수 없다. 컴플라이언스의 출발점은 **자산 식별(무엇이 있는가) →
분류(얼마나 중요한가) → 통제 차등 적용(중요할수록 강하게)**. ISMS-P/ISO 27001 모두 자산 관리가 기초 통제.

```
 식별(Inventory) → 분류(Classification) → 소유자 지정 → 통제 차등
 "있는 줄 몰랐던 자산"이 가장 위험(shadow IT, 방치 서비스)
```

---

## 2. 자산 식별 — 무엇이 있는가

```bash
# 웹 애플리케이션 자산(vhost) 목록
docker exec el34-web sh -c "ls /etc/apache2/sites-enabled/"
```
- el34-web: 다수 vhost(dvwa, juice, admin, ai 등) = 각각이 보호 대상 자산. 계정·데이터·네트워크 서비스도
  자산. 누락 없이 대장(臺帳)에 등록.

---

## 3. 자산 분류 — 중요도 등급

CIA(기밀성·무결성·가용성) 영향으로 등급을 매긴다.

```
 등급 예: 1급(극비/핵심) · 2급(기밀) · 3급(내부) · 4급(공개)
 평가축: 기밀성(노출 시 피해) × 무결성(변조 시 피해) × 가용성(중단 시 피해)
```
- 예) 결제 DB = 기밀 H·무결 H·가용 H → 1급. 공개 카탈로그 = 가용 위주 → 3~4급.

---

## 4. 데이터 분류

```
 공개(Public) · 내부(Internal) · 기밀(Confidential) · 극비(Restricted)
```
- 개인정보·카드정보·인증정보는 최소 기밀 이상. 분류 라벨이 암호화·접근통제 수준을 결정.

---

## 5. 자산 대장(Inventory)과 소유자

```
 자산명 · 유형 · 위치 · 소유자 · 중요도 · 데이터분류 · 통제수준
```
- **소유자(Owner)** 지정이 핵심 — 책임 주체 없는 자산은 방치된다. CMDB로 최신성 유지.
