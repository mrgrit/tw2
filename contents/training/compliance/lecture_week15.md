# 컴플라이언스 W15 — 기말: 종합 컴플라이언스 감사

> 컴플라이언스 트랙 15주차 (기말 평가). 선행: W01–W14. 인프라: el34. 플랫폼: tw2.
> 표준: ISMS-P / CIS / PCI-DSS 종합.

---

## 1. 이 시험이 평가하는 것

한 학기(W01~W14)의 전 영역을 **하나의 시스템에 대한 종합 컴플라이언스 감사**로 통합한다 — 정보노출·
접근통제·암호화·로깅·구성평가·무결성·취약점·개인정보를 한 바퀴 점검하고, **준수율(점수)·갭·시정
로드맵**을 갖춘 감사 보고서를 산출한다.

```
 종합 감사 흐름
 정보노출 → 접근통제 → 암호화 → 로깅 → 구성평가(SCA) → 무결성(FIM) → 취약점 → 개인정보
 → 준수율 산정 → 갭 + 시정 로드맵(우선순위)
```

---

## 2. 평가 항목 (전 영역 한 바퀴)

| 영역 | 점검 | 기대 |
|------|------|------|
| 정보노출(W01) | ServerTokens | 갭(Prod 아님) |
| 접근통제(W02·04) | PermitRootLogin/암호정책 | 혼합(root 차단·만료 갭) |
| 암호화(W05) | TLS | 준수(TLSv1.3) |
| 로깅(W06) | Wazuh | 준수 |
| 구성평가(W09) | SCA | 가동 |
| 무결성(W10) | FIM | 가동 |
| 취약점(W07) | nuclei | 발견 |
| 개인정보(W12) | Feedbacks | 갭(노출) |

---

## 3. 실습(기말 lab) 형식 — 10 미션

1. **점검**: 대상
2. **정보노출**: ServerTokens(갭)
3. **접근통제**: 암호 만료(갭)
4. **암호화**: TLS(준수)
5. **로깅**: Wazuh(준수)
6. **구성평가**: SCA(가동)
7. **무결성**: FIM(가동)
8. **취약점**: nuclei
9. **개인정보**: Feedbacks(갭)
10. **종합 감사 보고서**: 준수율+갭+시정 로드맵

> 명령은 el34 호스트(`ssh ccc@192.168.0.151`)에서 `docker exec el34-X` 로. 인가된 실습만.

---

## 4. 트랙 총정리

W01~W15로 컴플라이언스의 전 영역을 익혔다 — 프레임워크·거버넌스·자산·접근통제·암호화·로깅·취약점관리·
구성평가(SCA)·변경관리(FIM)·사고대응·데이터보호·공급망·감사. **통제의 존재를 증적으로 증명하고,
갭을 위험 기반으로 시정**하는 것이 컴플라이언스의 본질이다. 보안 점검(공격 관점)과 컴플라이언스(감사
관점)는 상호 보완하며, 둘을 함께 갖춘 조직이 진짜 안전하다.
