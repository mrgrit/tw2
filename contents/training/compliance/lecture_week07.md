# 컴플라이언스 W07 — 취약점 관리 컴플라이언스

> 컴플라이언스 트랙 7주차. 선행: W01–W06. 인프라: el34. 플랫폼: tw2. 표준: PCI-DSS 11·6, ISMS-P 2.11.

---

## 1. 이번 주의 통찰 — 취약점은 줄어들지 않는다, 관리될 뿐

매일 새 CVE가 공개된다. 한 번의 점검으로 끝나지 않으며, **정기적으로 스캔하고, 위험도별로 패치하고,
재검증하는 순환 프로세스**가 컴플라이언스의 요구다. PCI-DSS 11.3(정기 스캔)·6.3(패치 관리).

```
 취약점 관리 순환
 식별(스캔) → 평가(위험도/CVSS) → 조치(패치/완화) → 재검증(재스캔) → 반복
```

---

## 2. 정기 스캔 (PCI-DSS 11.3)

```bash
nuclei -u http://10.20.30.1 -H 'Host: juice.el34.lab' \
  -t /root/nuclei-templates/http/misconfiguration/http-missing-security-headers.yaml -silent -nc
```
- PCI: **분기별 + 중대한 변경 후** 스캔 의무. 내부/외부 스캔 분리. el34는 nuclei로 자동 스캔(W13).

---

## 3. 결과 분류 (위험도)

```
 nuclei severity: critical / high / medium / low / info
 → CVSS와 연계해 위험도 등급 부여(W14), 패치 우선순위 결정
```

---

## 4. 패치 관리 (PCI-DSS 6.3)

```
 위험도별 패치 SLA: Critical 즉시(예 ≤7일) · High 30일 · Medium 90일
 변경관리 절차(테스트→승인→배포→검증)를 거쳐 적용
 패치 불가 시 보상 통제(WAF 가상패치 등) 문서화
```

---

## 5. 정기성·재검증

```
 정기 스캔(분기) + 변경 후 스캔 + 패치 후 재스캔으로 조치 확인
 미조치 항목은 예외 승인(위험 수용) + 재검토 일정 등록
```

---

## 6. 실습(lab) 형식 — 8 미션

1. **점검**: 대상
2. **정기 스캔**: nuclei (PCI 11.3)
3. **결과 분류**: severity
4. **패치 관리**: 위험도별 SLA
5. **취약점 관리 순환**: 식별~재검증
6. **보상 통제**: 가상패치
7. **방어**: 취약점 관리 프로세스
