# W15 — 기말: 한 APT 캠페인을 5종 장비 + 위협인텔로 끝까지 막아내기 (수료)

> 보안운영 트랙 15주차 (수료 시험). 선행: W01–W14 전부. 인프라: el34. 플랫폼: tw2.

---

## 1. 수료 시험 — 14주의 무기를 하나의 캠페인에

지난 14주 동안 방화벽·IDS·WAF·SIEM·osquery·sysmon·위협인텔을 하나씩 익혔다. 기말은 **한 APT 그룹**이
**5단계 킬체인**으로 들어오고, 너는 **모든 무기를 하나의 캠페인에 총동원**해 각 단계를 탐지·차단·헌팅하고
APT IR 보고서로 종합한다.

```
 APT 킬체인 5단계                         동원 무기 (배운 주차)
 ──────────────────                      ──────────────────────
 ① 정찰(Recon)              →  IDS(W03/04) + WAF scanner(W05)
 ② 웹 익스플로잇 침투       →  WAF anomaly 949110 차단(W05/08)
 ③ 호스트 발판·측면이동     →  osquery 헌팅(W06/07/14)
 ④ C2 채널·유출            →  sysmon(W11) + osquery sockets + 인텔 룰(W12/13)
 ⑤ 대응·복구(IR)           →  SIEM 수렴(W08/09/10) + APT IR 보고서
```

핵심 평가: **단일 무기 실력이 아니라, 한 캠페인을 전 계층에서 추적·대응하는 통합 능력.**

---

## 2. APT란 — 끈질긴 표적 공격

APT(Advanced Persistent Threat)는 단발 공격이 아니라 **목표를 정해 끈질기게**(정찰→침투→지속성→
측면이동→유출) 단계적으로 진행한다. 그래서:
- **한 단계만 막으면 부족하다** — 다른 단계로 우회한다. 다층 방어가 필수.
- **출처 IP·시각으로 단계를 한 캠페인으로 엮어야** 한다(el34는 출처 보존 → 상관 가능).
- **IOC를 인텔로 격상**해 재발/변형을 자동 탐지해야 한다.

이 시험의 APT는 외부 공격자(10.20.30.202)가 일관된 출처로 5단계를 진행한다.

---

## 3. 단계별 대응 — 어느 무기로 끊나

### ① 정찰 → IDS + WAF
스캐너 UA(sqlmap/nikto)로 표면을 훑는다. Suricata 룰(sid 9015001)로 네트워크에서, WAF 913으로 HTTP에서.

### ② 웹 익스플로잇 → WAF anomaly
SQLi로 침투. ModSec anomaly 누적(942→949110)으로 403 차단(dvwa 차단 모드).

### ③ 호스트 발판 → osquery 헌팅
침투 후 백도어 계정(w15ghost) + C2 리스너(61515). 네트워크가 못 보는 호스트 내부를 osquery로 사냥.

### ④ C2/유출 → sysmon + 인텔 룰
인코딩 C2 명령·외부 연결은 sysmon 이벤트로, APT의 IOC(C2 포트/도구)는 Wazuh 인텔 룰(id 101510)로 격상.

### ⑤ 대응·복구 → SIEM + IR 보고서
5계층 흔적이 Wazuh로 수렴. 출처 IP로 한 캠페인 타임라인을 그리고 APT IR 보고서로 종합한다.

---

## 4. APT IR 보고서 — 5단계 종합

수료의 마무리는 **APT IR 보고서**다. 단계별로:

| 단계 | 공격 | 끊은 무기 | 증거 |
|------|------|-----------|------|
| ① 정찰 | 스캐너 UA | IDS(9015001) + WAF(913) | eve.json sig / audit 913 |
| ② 침투 | SQLi | WAF(942→949110) | audit 403 + remote 10.20.30.202 |
| ③ 발판 | w15ghost / 61515 | osquery(users/listening_ports) | uid / pid:port |
| ④ C2 | 인코딩/연결 | sysmon(1/3) + 인텔 룰 101510 | EventID / logtest |
| ⑤ 대응 | — | SIEM 수렴 + 출처 상관 | alerts.json 다계층 |

이 표를 증거와 함께 채울 수 있으면 보안운영 과정을 **수료**할 자격이 있다.

---

## 5. 실습(기말 lab) 형식 — 12 미션

1. **점검**: 전 무기(fw/ips/web/host/sysmon/siem/intel) 가용성
2. **① 정찰 재현**: 스캐너 UA → WAF 913
3. **① IDS 룰(sid 9015001)**: 정찰 탐지 → reload → eve → cleanup
4. **② 웹 침투 재현**: SQLi → 403
5. **② WAF 차단**: 942 → 949110 anomaly
6. **③ 호스트 발판 재현**: C2 리스너 61515 + 계정 w15ghost
7. **③ osquery 헌팅**: 발판 사냥
8. **④ C2 sysmon**: 인코딩/연결 이벤트
9. **④ APT 인텔 룰(id 101510)**: IOC 격상 → logtest
10. **⑤ SIEM 수렴 + 출처 상관**: 5계층 한 타임라인
11. **⑤ APT IR 보고서**: 5단계 종합
12. **정리 확인**: 모든 실습 흔적 0 (공유 인프라 보존)

> 모든 명령은 el34 호스트(`ssh ccc@192.168.0.151`)에서 `docker exec el34-<comp>`(sysmon은 호스트 syslog).
> IDS/Wazuh 룰은 추가→검증→삭제, 발판은 self-clean — 공유 인프라를 베이스 상태로 되돌린다.

## 6. 수료를 축하한다

15주 동안 너는 한 침입을 **방화벽에서 막고, IDS로 탐지하고, WAF로 차단하고, osquery/sysmon으로
호스트를 사냥하고, Wazuh로 한 평결로 모으고, 위협인텔로 격상하고, 능동적으로 헌팅하는** 보안운영의
전 주기를 익혔다. 이제 너는 한 캠페인을 전 계층에서 끝까지 막아내는 **보안 운영자**다. 🎓
