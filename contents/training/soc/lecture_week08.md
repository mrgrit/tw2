# SOC W08 — 중간고사: 흩어진 로그를 교차 분석해 하나의 공격 서사로 종합

> SOC 관제 트랙 8주차 (중간 평가). 선행: W01–W07. 인프라: el34. 플랫폼: tw2.

---

## 1. 이 시험이 평가하는 것

지난 7주의 SOC 분석 역량을 종합한다 — 트리아지(W01), 인증 로그(W02), 웹 교차(W03), 커스텀 룰(W04),
경보 관리(W05), ATT&CK 캠페인(W06), SIGMA(W07). 중간고사는 **흩어진 로그를 교차 분석해 하나의 공격
서사(narrative)로 종합**하는 분석가의 종합 능력을 본다.

```
 인증 로그(auth.log)  ─┐
 웹 로그(ModSec/access)─┼─→ 교차 분석 → 한 공격자의 서사 →  우선순위 + 에스컬레이션
 네트워크(Suricata eve)─┤    (출처/시간/ATT&CK)
 SIEM(alerts.json)    ─┘
```

핵심: 한 로그만 보면 단편. **교차로 봐야 "한 공격자가 정찰→침투→대입을 했다"는 서사**가 보인다.

---

## 2. 종합 분석의 순서

1. **수집**: 네 소스(인증/웹/네트워크/SIEM)에서 관련 이벤트를 모은다.
2. **상관**: 출발지 IP + 시간으로 묶는다(한 공격자인가?).
3. **매핑**: 각 단계를 ATT&CK 전술/기술로.
4. **서사**: 시간순으로 "무슨 일이 일어났나"를 한 문장씩.
5. **판정**: 우선순위(P1~P4) + 에스컬레이션 + 침투 성공 여부.

---

## 3. 공격 서사의 예

```
 T0  포트 스캔 (10.20.30.202)        ← Suricata eve, T1046 정찰
 T1  SQLi 시도 (같은 출발지)          ← ModSec 942/949110, T1190 침투
 T2  SSH 무차별 대입 (다발)           ← auth.log Invalid user, T1110 자격증명
 T3  (성공 흔적?) Accepted 유무       ← auth.log, 침투 성공 판정
 ─────────────────────────────────────
 결론: 한 공격자가 정찰→웹침투→인증공격. 성공 흔적 없으면 "시도(차단)", 있으면 "침해" P1
```

좋은 서사 = 출처로 묶이고, 시간순이며, ATT&CK 단계가 일관되고, 증거(로그)가 각 단계에 붙는다.

## 4. 실습(중간고사 lab) 형식 — 10 미션

1. **점검**: 네 분석 소스
2. **다단계 공격 재현**: 스캔 + 웹 + SSH 무차별 대입
3. **네트워크 분석**: Suricata eve
4. **웹 분석**: ModSec + access
5. **인증 분석**: auth.log
6. **SIEM 종합**: alerts.json 수렴
7. **교차 상관**: 출처/시간으로 묶기
8. **ATT&CK 서사**: 단계별 매핑 + 시간순
9. **우선순위/에스컬레이션**: P1~P4 + 침투 여부
10. **중간고사 종합 보고서**

> 명령은 el34 호스트(`ssh ccc@192.168.0.151`)에서. 관측·분석 종합(인프라 변경 없음).
