# SOC W13 — 인텔 주도 트리아지: '알려진 위협'이라는 한마디가 대응을 가속한다

> SOC 관제 트랙 13주차. 선행: W05(트리아지)·W11(IOC). 인프라: el34 (Wazuh + OpenCTI/MISP). 플랫폼: tw2.

---

## 1. 이번 주의 통찰 — 인텔이 트리아지를 가속한다

같은 경보라도 **"이 출발지/도구는 알려진 악성"** 이라는 위협 인텔 한마디가 붙으면 트리아지가 확 빨라진다.
조사할 것을 조사하기 전에 우선순위가 정해진다.

```
 인텔 없음:  경보 → 조사(누구지? 위험한가?) → 판정 → 우선순위  (느림)
 인텔 있음:  경보 + "known-bad" → 즉시 P1 격상 → 대응          (빠름)
```

SOC의 처리량(throughput)은 인텔로 결정된다 — 아는 위협은 빨리 쳐내고, 모르는 것에 시간을 쓴다.

---

## 2. 인텔 enrichment — 경보에 "맥락" 한 줄

경보의 출발지/도구/해시를 인텔과 대조해 맥락을 붙인다:
- **알려진 악성?** (CDB list / OpenCTI / MISP) → 즉시 격상.
- **평판/지역?** (W13 enrichment 개념) → 우선순위 조정.
```bash
# 알려진 IOC와 대조 (CDB list)
echo '{"tool":"sqlmap","src_ip":"10.20.30.202"}' | sudo /var/ossec/bin/wazuh-logtest   # 매칭 룰이 격상
```

---

## 3. 인텔 주도 트리아지 — 결정 가속

| 경보 | 인텔 | 트리아지 결정 |
|------|------|--------------|
| SQLi from X | X = known-bad(OpenCTI) | **즉시 P1**, 조사 생략 가능 |
| 스캔 from Y | Y = 평판 없음 | 일반 절차(조사 후 판정) |
| 로그인 from Z | Z = 내부 정상 | P4(오탐 가능) |

인텔이 "이미 아는 답"을 주면 분석가는 조사를 건너뛰고 대응으로 직행 → 처리량↑.

---

## 4. 인텔 운영 — CDB + OpenCTI/MISP

W11에서 추출한 IOC, W12-13의 인텔을 **CDB list**로 운영화해 경보에 자동 매칭(W12 secuops 패턴).
el34엔 OpenCTI/MISP가 가동 중 — 인텔 소스 → CDB → Wazuh 매칭 룰 → 자동 격상.
```bash
docker ps --format '{{.Names}}' | grep -iE 'opencti|misp' | head
```

## 5. 실습(lab) 형식 — 8 미션

1. **점검**: 인텔 스택(Wazuh + OpenCTI/MISP)
2. **공격 재현**: 알려진 악성 도구(sqlmap)
3. **인텔 없는 트리아지**: 일반 경보(조사 필요)
4. **인텔 매칭 룰(id 101310)**: known-bad → 격상(logtest)
5. **인텔 주도 격상**: 즉시 P1
6. **인텔 플랫폼**: OpenCTI/MISP
7. **가속 효과**: 처리량 향상
8. **인텔 트리아지 보고서**

> 명령은 el34 호스트(`ssh ccc@192.168.0.151`)에서. 매칭 룰은 logtest로만 검증 + 삭제(공유 SIEM 보존).
