# SOC W01 — 다중벡터 공격 vs SOC 관제·L1 트리아지

> SOC 관제 트랙 1주차. 인프라: el34 (Wazuh SIEM + Suricata/ModSec/osquery 소스). 플랫폼: tw2.

---

## 1. SOC란 — 경보의 강을 다루는 곳

SOC(Security Operations Center)는 쏟아지는 보안 이벤트를 **수집 → 분류(triage) → 우선순위 → 대응**으로
흘려보내는 곳이다. 핵심 도구는 SIEM(여기선 **Wazuh**) — 흩어진 소스의 이벤트를 한 곳으로 모은다.

```
 [소스]                    [SIEM = Wazuh]              [분석가]
 Suricata(IDS) ─┐
 ModSec(WAF)   ─┼─→  alerts.json (한 평결 스트림)  →  L1 트리아지
 osquery(host) ─┤                                      L2 심층분석
 sshd/syslog   ─┘                                      L3 IR/헌팅
```

이 주차는 SOC의 출발점 — "이벤트가 **어디서 와서 어떻게 모이는가**"와 "**무엇부터 볼 것인가**"다.

---

## 2. SOC 분석가 등급 — L1 / L2 / L3

| 등급 | 역할 | 이번 주 |
|------|------|---------|
| **L1** | 경보 수집·분류·1차 우선순위(triage). 대부분 여기서 끝(오탐/저위험) | ★ 이번 주 |
| L2 | L1이 올린 것 심층 분석·상관 | 이후 |
| L3 | IR·위협 헌팅·룰 개발 | 이후(W09+) |

L1의 무기는 **빠른 판단** — 경보 하나하나를 깊게 파지 않고, **출발지·시그니처·시각·심각도**로 분류해
"버릴 것 / 올릴 것"을 가른다.

---

## 3. L1 트리아지 4요소 — 경보에서 무엇을 보나

경보 하나를 받으면 분석가는 네 가지를 즉시 읽는다:

| 요소 | Wazuh 필드 | 의미 |
|------|-----------|------|
| **출발지(source)** | `.data.srcip` / `.data.src_ip` | 누가 (외부/내부, 평판) |
| **시그니처(what)** | `.rule.description` / `.rule.id` | 무슨 공격 |
| **시각(when)** | `.timestamp` | 언제 (업무시간? 야간?) |
| **심각도(level)** | `.rule.level` | 얼마나 위험 (Wazuh level 0–16) |

```bash
docker exec el34-siem sh -c 'tail -1 /var/ossec/logs/alerts/alerts.json | jq "{src:.data.srcip, sig:.rule.description, when:.timestamp, level:.rule.level, groups:.rule.groups}"'
```

## 4. 다중벡터 공격 — 한 공격자, 여러 갈래

능숙한 공격자는 한 갈래만 쓰지 않는다. 이번 주 공격자(외부 10.20.30.202)는:

```
 ① 포트 스캔(정찰)        → Suricata 스캔 탐지 (네트워크)
 ② SSH 무차별 대입        → Wazuh authentication_failed (인증 로그)
 ③ 웹 공격                → ModSec/Suricata (웹/네트워크)
```

SOC 화면엔 이 셋이 **서로 다른 경보**로 뜬다. L1의 진짜 실력은 이걸 **한 공격자의 단계적 행위**로
엮는 것(상관) — 출발지 IP가 같으면 한 캠페인이다(el34는 출처 보존 → 상관 가능).

---

## 5. 우선순위 — 무엇부터 보나

모든 경보가 같지 않다. L1은 빠르게 줄을 세운다:

| 우선순위 | 신호 | 예 |
|----------|------|----|
| **높음(P1)** | 침투 성공/높은 level/중요 자산 | 인증 성공 후 이상행위 |
| **중간(P2)** | 진행형 공격/반복 | SSH 무차별 대입 다발 |
| **낮음(P3)** | 정찰/단발/저level | 포트 스캔 1회 |

> 빈도(짧은 시간 다발)는 강한 신호다 — 스캔 1회(P3)와 무차별 대입 100회(P2)는 다르다.
> 무차별 대입은 "곧 침투를 시도한다"는 진행형 신호라 스캔보다 우선.

## 6. 실습(lab) 형식 — 8 미션

1. **SOC 대시보드 점검**: Wazuh가 경보 수집 중인가
2. **다중벡터 공격 재현**: 스캔 + SSH 무차별 대입 + 웹 공격
3. **트리아지 — 스캔 경보**: 출발지/시그니처/심각도 식별 → 정찰 분류(P3)
4. **트리아지 — 인증 실패 경보**: 빈도/계정으로 무차별 대입 판정 → P2
5. **트리아지 — 웹 공격 경보**: 시그니처/대상 식별
6. **우선순위 정렬**: 세 경보를 P1/P2/P3로
7. **상관(Purple)**: 같은 출발지로 한 캠페인 종합
8. **L1 트리아지 보고서**: 4요소 + 우선순위 + 상관

> 명령은 el34 호스트(`ssh ccc@192.168.0.151`)에서 `docker exec el34-<comp>` 로. 공격은
> `docker exec el34-attacker`, 분석은 `el34-siem`의 alerts.json. 관측·분석 중심이라 인프라 변경 없음.
