# SOC W12 — 내부 위협: 인가된 사용자의 권한 남용을 UEBA로 잡고 증거를 보전하기

> SOC 관제 트랙 12주차. 선행: W02(인증)·W09(IR). 인프라: el34. 플랫폼: tw2.

---

## 1. 이번 주의 통찰 — 가장 까다로운 위협, 내부자

외부 공격은 "들어오는" 걸 잡으면 된다. 내부 위협은 다르다 — **인가된 사용자**가 정상 권한으로 악행을
한다. 시그니처가 없다. 잡는 법은 **행위(behavior)** — 평소와 다른 패턴을 본다. 이것이 **UEBA**
(User and Entity Behavior Analytics)다.

```
 외부 위협: 시그니처(알려진 악성)         내부 위협: 행위 편차(baseline 대비 이상)
 ──────────────────────             ─────────────────────────────
 "이 IP/도구는 악성"                 "이 사용자가 평소와 다르게 행동한다"
                                      (야간 로그인 / 권한 상승 / 대량 접근 / 데이터 staging)
```

---

## 2. UEBA — 행위 baseline과 편차

평소 행동(baseline)을 알아야 이상을 안다.

| 행위 축 | 정상 | 이상(내부 위협 신호) |
|---------|------|---------------------|
| **로그인 시각** | 업무시간 | 야간/주말 |
| **권한 사용** | 평소 명령 | 갑작스런 sudo 남용 |
| **파일 접근** | 담당 범위 | 대량/민감 파일 접근 |
| **데이터 이동** | 없음 | staging(압축)/유출 |

el34에서 행위 소스: 호스트 `auth.log`(로그인/sudo) + osquery(파일/프로세스).

---

## 3. ① Identify — 비정상 행위

```bash
# sudo 사용 패턴 (권한 남용 신호)
grep -aE "sudo:.*COMMAND" /var/log/auth.log | tail -10
# 로그인 시각 (야간 로그인?)
grep -a "Accepted" /var/log/auth.log | tail -10
```
- 갑작스런 sudo 다발, 야간 로그인, 평소 안 쓰던 명령 = 행위 편차.

---

## 4. ② Hunt — 권한 남용 + 유출 흔적

```bash
# 데이터 staging (민감 파일 압축 = 유출 준비)
docker exec el34-web osqueryi --json 'SELECT path,size FROM file WHERE path LIKE "%exfil%" OR path LIKE "/tmp/%.tar.gz";'
```
- staging 아카이브(/tmp/*.tar.gz of /etc 등) = 유출 직전 신호.
- 대량 파일 읽기 + 압축 + 외부 전송 = 유출 체인.

---

## 5. ③④⑤ Contain → Eradicate → Recover (증거 보전 우선)

내부 위협은 **법적 대응**이 따르므로 **증거 보전**이 최우선:
- **증거 보전**: 로그/파일을 변경 없이 보존(타임스탬프, 해시). 함부로 지우지 말 것.
- **Contain**: 유출 채널 차단(sid 9512001), 계정 권한 동결(잠금).
- **Eradicate/Recover**: 법무 협의 후 비인가 접근 제거 + 복구.
- 내부 위협 IR은 **HR/법무 협업** — 기술 대응과 별개로 절차를 따른다.

---

## 6. 실습(lab) 형식 — 8 미션

1. **점검**: 행위 소스(auth.log/osquery)
2. **내부 위협 재현**: sudo 남용 + 데이터 staging
3. **UEBA Identify**: 비정상 행위(sudo/시각)
4. **Hunt**: 권한 남용 + staging 흔적
5. **증거 보전**: 변경 없이 보존(해시)
6. **Contain**: 유출 채널 격리(sid 9512001)
7. **Recover**: 비인가 접근 제거 + 복구
8. **내부 위협 IR 보고서**(증거 보전·법무)

> 명령은 el34 호스트(`ssh ccc@192.168.0.151`)에서. staging/룰은 self-clean(공유 인프라 보존).
