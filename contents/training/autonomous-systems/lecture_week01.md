# autonomous-systems W01 — CPS 보안 개론: 사이버물리시스템과 위협 모델

> **본 주차의 한 줄 요약**
>
> autonomous-systems는 **자율 시스템**(드론·자율주행차·로봇·산업 제어)의 보안을 다룬다. 이들의 공통 본질은
> **CPS(Cyber-Physical System, 사이버물리시스템)** — **사이버(소프트웨어·네트워크·AI)** 와 **물리(센서·구동기·
> 실제 움직임)** 가 결합된 시스템이다. IoT가 "연결된 장치"라면, CPS는 한 걸음 더 나아가 **스스로 판단해 물리
> 세계를 움직인다**(드론이 날고, 차가 달리고, 로봇이 작동). CPS 보안의 핵심 특수성: **사이버 공격이 곧 물리
> 결과**가 된다 — 소프트웨어 취약점 하나가 데이터 유출이 아니라 **드론 추락·차량 사고·로봇 오작동** 같은 물리
> 재앙·인명 피해로 이어진다. 그래서 CPS 위협 모델은 세 계층을 본다: ① **사이버 계층**(SW·통신·AI 모델 공격),
> ② **물리 계층**(센서 스푸핑·구동기 조작), ③ **브리지**(사이버→물리 변환 지점, 가장 위험). 공격자는 사이버로
> 진입해 물리 결과를 노리거나(드론 하이재킹), 물리 계층을 직접 속인다(GPS 스푸핑·센서 교란). 방어의 최우선은
> **안전(Safety)** — 보안이 뚫려도 물리적으로 안전해야 한다(독립 안전 장치·페일세이프). CPS는 자율성이 커질수록
> 위험도 커지는, 사이버 보안의 최전선이다.
>
> **한 줄 결론**: CPS(사이버물리시스템)는 사이버와 물리가 결합돼 **사이버 공격이 곧 물리 결과**가 된다. 위협
> 모델은 사이버·물리·브리지 3계층을 보고, 방어는 **안전 최우선**(페일세이프·독립 안전 장치)이다.

## 학습 목표

본 주차 종료 시 학생은 다음 5가지를 **본인 손으로** 할 수 있어야 한다.

1. **CPS**와 IoT의 차이(자율성·물리 결과)를 설명한다.
2. CPS를 **사이버·물리·브리지** 계층으로 매핑한다(CPS_MAPPED).
3. **사이버→물리 공격 경로**를 구성한다(PHYSICAL_IMPACT).
4. **안전 필수(safety-critical) 위험**을 평가한다(SAFETY_RISK).
5. 왜 CPS는 안전이 최우선인지 설명한다.

> **이 주차의 시선** — 사이버와 물리가 만나는 CPS의 특수 위험과 안전 최우선 방어를 세운다.

---

## 0. 용어 해설 (CPS)

| 용어 | 영문 | 뜻 | 비유 |
|------|------|----|------|
| **CPS** | Cyber-Physical System | 사이버물리시스템 | 생각하는 기계 |
| **사이버 계층** | Cyber Layer | SW·통신·AI | 두뇌·신경 |
| **물리 계층** | Physical Layer | 센서·구동기 | 눈·손발 |
| **브리지** | Bridge | 사이버↔물리 변환 | 감각-운동 접점 |
| **페일세이프** | Fail-safe | 안전 실패 | 비상 정지 |

> **헷갈리기 쉬운 한 쌍** — *IoT* 는 "연결된 장치(데이터)", *CPS* 는 "스스로 판단해 물리를 움직임(행동)"이다.
> CPS는 뚫리면 물리 결과.

## 0.5 신입생 친화 핵심 개념

### 0.5.1 CPS — 사이버와 물리의 결합

```mermaid
graph LR
    S["센서(물리)"] --> C["사이버<br/>(SW·AI 판단)"]
    C --> A["구동기(물리)"]
    A --> W["실제 움직임<br/>(비행·주행·작동)"]
    ATK["공격자"] -.사이버 또는 물리 진입.-> C
    ATK -.센서 스푸핑.-> S
    style W fill:#f85149,color:#fff
    style ATK fill:#f85149,color:#fff
```

센서로 세계를 인식→사이버가 판단→구동기로 물리 행동. 이 고리 어디를 공격해도 **물리 결과**가 나온다.

### 0.5.2 사이버 공격 = 물리 결과

CPS의 근본 특수성: 소프트웨어 취약점이 **물리 재앙**이 된다. 드론 제어 하이재킹→추락, 자율주행 모델 속이기→
사고, 로봇 명령 주입→오작동, PLC 조작→폭발(Stuxnet). 데이터 손실이 아니라 **생명·안전**이 걸린다. 이것이 CPS
보안을 특별하게 만든다.

### 0.5.3 3계층 위협 모델

- **사이버 계층**: SW 취약점·통신 감청/주입·AI 모델 공격(적대적 입력, W07·W13).
- **물리 계층**: 센서 스푸핑(GPS·라이다·카메라)·구동기 직접 조작·RF 교란.
- **브리지**: 사이버 명령이 물리 행동이 되는 지점 — 여기가 뚫리면 사이버가 물리를 지배. 가장 위험.

### 0.5.4 안전 최우선 — 페일세이프

CPS 방어의 최우선은 **안전(Safety)**. 보안이 뚫려도 **물리적으로 안전**해야 한다: 독립 안전 장치(SW와 분리),
페일세이프(이상 시 안전 정지·귀환), 물리적 한계(지오펜싱·속도 제한). 보안(사이버)과 안전(물리)은 다르지만
CPS에선 함께 간다 — 보안 실패가 안전 실패가 되지 않게.

### 0.5.5 el34 맥락과 한계

CPS(드론·차량·로봇)는 실물 하드웨어가 필요하다. 본 과목은 **위협 모델·공격 경로·방어 로직**을 결정론 시뮬과
GPU 분석으로 익히고, 물리 실행(RF·센서·구동)은 실물 하드웨어가 필요함을 명시한다. 개념·분석·방어 설계 능력에
집중한다.

## 2. 흔한 오해·관제자 노트

- **"CPS는 IoT와 같다"** — CPS는 자율 판단+물리 행동. 뚫리면 물리 재앙.
- **"보안만 하면 안전"** — 보안≠안전. 보안 뚫려도 안전하게(페일세이프).
- **"SW 버그는 데이터 문제"** — CPS에선 물리 사고·인명. 안전 최우선.
- **관제 관점** — CPS가 사이버·물리·브리지 3계층에서 평가됐는지, 페일세이프·독립 안전 장치가 있는지, 사이버
  침해가 물리 안전으로 번지지 않는지 점검한다. CPS는 안전 최우선.
